# Network

이번에는 네트워크를 공부하면서 이해한 내용을 공유하고 기록하고자 포스팅합니다. 조금씩 네트워크를 이해할수록 이전에 했던 삽질에 대해 깨달음을 얻고 있습니다.

네트워크를 둘 이상의 컴퓨터와 이들을 연결하는 링크의 조합이라는 개념을 가지고 있습니다. 연결된 컴퓨터가 정보를 교환하기 위해 통신 프로토콜을 가지고 서로 다른 컴퓨터와 정보를 전송하는 것을 네트워킹이라고 합니다.

그렇다면 어떻게 데이터를 주고 받고 하는 걸까요?

## IP Address

ip는 "인터넷 프로토콜 주소"를 줄여서 표기한 이름입니다. 컴퓨터 네트워크에서 장치를 식별하기 위해 사용됩니다. 실생활의 예를 들어, 집 주소지가 물리적으로 존재하는 부동산의 지번 또는 도로명의 위치 이름입니다.

ip는 4개의 octet으로 이루어졌습니다. 1octet당 8bit이고, 총 4 \* 8 = 32bit 입니다. ip는 몇개까지 생성가능할까요?

최대 2의 32제곱으로 4,294,967,296개 가질 수 있습니다. 단, ipv4 32bit 주소 체계만 보았을 때 입니다.

### IP Address Class

IP주소 클래스는 네트워크 구성을 나타내는 데 사용되는 IP주소 범위 입니다. 기본적으로 5개의 클래스가 있고, A, B, C만 알고 있어도 충분할 것이라 생각합니다.

| class | 범위      | network                   | host              |
| ----- | --------- | ------------------------- | ----------------- |
| A     | 0xxx xxxx | 0~127                     | 0.0.0~255.255.255 |
| B     | 10xx xxxx | 128.0~191.255             | 0.0~255.255       |
| C     | 110x xxxx | 192.0.0.~223.255.255      | 0~255             |
| D     | 1110 xxxx | 224.0.0.0~239.255.255.255 | -                 |
| E     | 1111 xxxx | 240.0.0.0~255.255.255.255 | -                 |

#### A Class

A 클래스의 네트워크 영역은 0~127입니다. 호스트 영역은 나머지 24bit이고 할당 가능한 호스트 IP는 2의 24제곱인 16,777,216개 입니다. 총 IP의 개수는 128개의 네트워크에 호트스 IP 개수를 곱한 값인 2,147,483,648이 되겠습니다.

이때 네트워크의 비트수가 8bit이고 ip주소를 표기할 때 한 번쯤을 봤을 `/8`로 표시한다. 예를 들어, `116.82.27.6 /8`이라고 표기하게 된다.

이렇게 표기되는 서브넷 마스크는 `255.0.0.0`이 된다.
