# 서브넷마스크와 서브넷팅

사내 백엔드가 혼자이다 보니 이것저것 알아야 할 부분이 하나 둘이 아닌 걸 많이 느낍니다. 스위치도 제대로 몰랐던 입사 초기때와 달리 어느정도 원하는 구상대로 서버 환경을 구축하고 포트포워딩하거나 ip 액세스를 제어하는 등의 기초적인 설정을 할 수 있을 정도가 되었습니다.

무턱대고 시작한 네트워크 부분이지만 이론을 알고 있느냐, 그건 또 자신이 없습니다. 네트워크를 하나 씩 만져가며 실수도 해보고, 작동하는 흐름상 이렇게 되겠지 하는 감으로 여기까지 쌓아온거라 이론을 알면 좀 더 매끄럽겠다 싶어 이론을 병행해야겠다는 생각을 했습니다.

물론 아무것도 모르고 부딪힌 것은 아니지만 이론을 보면 볼수록 더 많은 것을 알아야겠다는 욕심도 생깁니다.

## 서브넷마스크

리눅스 환경에서 커널을 사용하다보면 자주 ip를 확인하는 일이 생깁니다. 혹은 사용자 cpu사용량 체크를 위해 top을 사용한다거나 route를 프린트하는 등의 작업이지요.

이때 자주 등장하는 녀석이 subnetmask였습니다. 궁금하지만 몰라도 크게 문제가 없었기에 지나치고 있었습니다. 이번에는 subnetmask를 알아보고 정리하고자 합니다.

먼저, IP와 Network에 대해 간단하게 짚고 넘어가겠습니다.

### IP

ip는 ip address라고도 하지요. 즉, ip는 주소를 나타내는 8비트 2진수가 4부분으로 이루어진 총 32비트 이진수로 구성이 된 수 입니다.

흔히 보는 `xxx.xxx.xxx.xxx` ip 포멧은 `.`으로 나눈 4개의 영역에 8비트 2진수가 각각 10진수로 변환된 수 입니다.

각 영역을 `Octet`이라고 합니다. 즉, 4개의 `Octet`이 있다는 말이기도 합니다. 하나의 네트워크 안에 IP들은 `네트워크 영역`이 같아야하고, `호스트 IP`는 서로 달라야 통신이 가능합니다.
